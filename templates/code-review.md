## 9. 问题处理策略

| 问题状态 | 含义 | 后续动作 |
| -------- | ---- | -------- |
| Open | 已发现但未处理 | 返回 /sfk-dev 修复 |

| Verified | 已复核确认修复 | 可进入后续阶段 |
| Deferred | 延后处理 | 必须记录风险和接受条件 |
| Accepted | 明确接受风险 | 必须记录接受人和原因 |
| Rejected | 判定不是问题 | 必须记录判定依据 |

## 11. 修复回流与再审要求

代码审查阶段不得直接修改业务源码。若审查结果为 changes_required 或 blocked，必须返回 /sfk-dev 处理修复；修复前仍需通过 `python scripts/sfk.py implementation approve development --summary "..."` 记录源码实现授权，修复后必须重新执行 /sfk-code-review。

| 触发条件 | 回流动作 | 再审要求 |
| -------- | -------- | -------- |
| changes_required | 返回 /sfk-dev 修复问题 | 修复后重新审查 |
| blocked | 返回上游阶段解除阻塞 | 阻塞解除后重新审查 |
| pass_with_risks | 记录风险接受或延后处理依据 | 测试阶段重点覆盖 |
| pass | 可进入 /sfk-test | 无额外要求 |
